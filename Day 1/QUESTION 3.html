DATATYPES

    The Datatypes in javascript are basically divided into two main types :
  • PRIMITIVE DATATYPES : These datatype can store only single value one at at a time
    This datatype is stored in STACK(memory allocation).

    (1.) Number : Stores the integer/numeric value.
    eg. var num = 100 //here variable num stored a numeric value.
    console.log(num);// O/P = 100

    (2.) String : String consists of sequence of characters.
    eg. var name = "LETS UPGRADE" // the variable name has stored the characters.
    console.log(name); // O/P = 'LETS UPGRADE'

    (3.) Boolean value : this datatype stores the value as 'true'/'false'.
    eg. var val = 'true' // val stores the boolean value.
    console.log; // O/P = true

    (4.) Bigint : In Javasacript by using the number datatype one can print only values upto 2^(53-1).
    The BigInt is a primitive datatype format which allows you to enter numbers larger than 2^(53-1).
    Whenever you need to print a large value BigInt is used.
    Now this datatype is used by appending n to the integer literal or by calling the function BigInt().
    eg. let value = 12345689n;
    console.log(value); // O/P = 12345689n

    (5.) Null : In JavaScript null is "nothing". It is supposed to be something that doesn't exist.
    Unfortunately, in JavaScript, the data type of null is an object.
    Its a bug in JS due to which the null is considered as an object.
    It should be null.You can empty an object by setting it to null.
    eg. let abc = {1,2,3};
    abc = null;//now the value is null yet the datatype is still an object.
    console.log(abc); // O/P = null

    (6.) Undefined : When a variable in JS does not have a value it is known as undefined. The type will be also undefined.
    eg. var name ; // here the value remains undefined.

  • NON-PRIMITIVE(Reference datatypes) : These are also known as composite datatypes.
    These are stored in HEAP(memory allocation).

    (1.) Object datatype : Object is a collection of data.
    An object contains properties, defined as a key-value pair. A property key (name) is always a string, but the value can be any data type, like strings, numbers, booleans, or complex data types like arrays, function and other objects.
    eg. let course = { name:'JavaScript',batch: 1, duration: '7 days'}
    console.log(course); // O/P = { name:'JavaScript',batch: 1, duration: '7 days'}

    (2.) Function datatype : Function is a callable object that executes a block of code.
    Functions can be stored in variables, objects, and arrays.
    Functions can be passed as arguments to other functions, and functions can be returned from functions.
    eg. var greeting = function(){
    return "Hello World!"}
    alert(typeof greeting) // O/P = function
    alert(greeting()); // O/P = Hello World! I

    (3.) Array datatype : The array object stores multiple values in a single variable.
    The values stored inside an array are called as "elements".
    Each element acquires a numeric position known as "index".
    The index in array starts from 0, that means the first element will be
    indexed as [0] ,second as [1] and so on.
    eg. let array = [2,4,6,8,10,12,14];
    console.log(array); // O/P = (7) [2,4,6,8,10,12,14]
    console.log(array[3]); // O/P = 8
